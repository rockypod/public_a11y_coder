This violates **SC 1.3.1 Info and Relationships** and **SC 3.3.2 Labels or Instructions**.

**Why**: Screen readers rely on programmatic associations between labels and inputs. Without them, users may not know which label applies to which field.

**Fix** (Drupal best practice):
1. Use the `#title` property in render arrays — Drupal auto-wraps in `<label for="...">` when `#title_display` is not set to `invisible`.
2. For custom HTML, ensure:
   ```twig
   <label for="email">Email address</label>
   <input id="email" name="email" type="email">
   ```
3. If using placeholders instead of labels, add `aria-label` or `aria-labelledby` to the input:
   ```twig
   <input aria-label="Email address" type="email">
   ```

**Note**: Visible labels alone are insufficient if not programmatically linked.
